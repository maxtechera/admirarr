admirarr: menos paneles, más control sobre tu media stack

Por qué lancé un CLI para ver estado, correr diagnósticos y rehacer partes críticas del stack Plex + Arr sin saltar entre paneles, logs y contenedores.

Ver el repo Ver instalación

Si corrés Plex, Radarr, Sonarr, Prowlarr y un downloader, ya sabés que el problema rara vez es tener el stack andando. El problema aparece después, cuando algo deja de responder y empezás a perseguir contexto entre interfaces, logs y contenedores.

Ahí nació admirarr. No como otra UI para sumar a la pila, sino como una capa operativa para recuperar contexto rápido. La apuesta es simple: si el stack ya está distribuido entre varias herramientas, el diagnóstico y la operación cotidiana no deberían estarlo.

El problema no es instalar un homelab, es operarlo bien

La instalación suele ser la parte más vistosa. Elegís servicios, configurás Docker, conectás volúmenes e indexers y en algún momento todo arranca. Pero esa sensación dura poco si la operación diaria sigue siendo fragmentada.

Con el tiempo, cualquier media stack gana complejidad. Aparecen automatizaciones parciales, rutas frágiles, credenciales repartidas y servicios que parecen sanos aunque el flujo completo esté roto. Cada panel muestra una parte de la verdad, pero casi nunca la película completa.

Ese fue el patrón que me cansó. No faltaban herramientas. Faltaba una forma de coordinar observación y acción sin pagar el costo de cambiar de contexto todo el tiempo.

Qué quise construir con admirarr

admirarr es un CLI para operar un stack Plex o Jellyfin más herramientas Arr con menos fricción. No intenta reemplazar Radarr, Sonarr o Prowlarr. Intenta darte una interfaz operativa consistente para tres momentos concretos.

admirarr status para tener una foto real del stack

Cuando algo parece raro, lo primero no es cambiar nada. Lo primero es entender. status junta servicios, librerías, descargas, disco y requests en una sola salida para que puedas decidir rápido dónde profundizar. Esa foto inicial reduce fricción cognitiva y evita abrir cinco tabs antes de tener una hipótesis.

admirarr doctor para diagnosticar antes de improvisar

Muchas fallas en un homelab no son misteriosas. Son repetitivas. Contenedores a medias, puertos caídos, paths inconsistentes, VPNs que parecen arriba pero no sirven, importaciones frenadas. doctor existe para convertir esos patrones en checks explícitos y resultados accionables.

Eso importa porque en self-hosting el costo más común no es una caída total. Es una hora perdida persiguiendo síntomas sueltos. Si la herramienta concentra checks y devuelve causas probables, la operación deja de sentirse como una colección de excepciones.

admirarr setup para reconstruir con menos dolor

A veces no querés arreglar. Querés rehacer una parte del sistema con menos pasos manuales. setup está pensado desde esa realidad, como una forma de converger más rápido hacia una configuración sana cuando inspeccionar ya no alcanza.

Por qué preferí una CLI y no otra dashboard web

Una CLI bien diseñada tiene dos ventajas fuertes. La primera es obvia: es rápida, scriptable y fácil de integrar. La segunda es más interesante: obliga a diseñar una superficie clara tanto para humanos como para máquinas. Si un comando tiene que ser legible para una persona y útil para un agente, el producto mejora.

Por eso el repo también publica archivos como SKILL.md y AGENTS.md, además de una página de instalación rápida. admirarr está pensado como interfaz humana e interfaz de agente al mismo tiempo.

El costo oculto de saltar entre paneles

En stacks distribuidos, el problema no es solo la cantidad de interfaces. Es el cambio de contexto. Cada cambio te obliga a recordar qué estabas validando, qué hipótesis seguía viva y qué parte del sistema ya habías descartado. Ese costo no aparece en ninguna dashboard, pero se siente cada vez que una falla simple se convierte en media hora perdida.

La tesis de admirarr no es prometer magia. Es reducir ese costo. Menos paneles no significa menos información. Significa una mejor secuencia para llegar a la información correcta.

Para quién está hecho

No está pensado para quien quiere tocar una vez un stack de Plex y olvidarse. Está pensado para la persona que ya vive con ese sistema, lo ajusta, lo observa y necesita volver a ponerlo en pie sin drama innecesario.

Si corrés un homelab en serio, seguramente ya desarrollaste tus propios rituales: revisar descargas, validar salud, mirar requests, inspeccionar contenedores, entender por qué una película no apareció o por qué una serie quedó en cola. admirarr intenta condensar esos rituales en comandos consistentes.

Qué no intenta hacer

También me importaba dejar un límite claro. admirarr no intenta convertirse en la única interfaz del stack. No quiere reemplazar las UIs donde sí tiene sentido profundizar. Tampoco intenta esconder la complejidad real detrás de una capa simplificada que se rompe cuando el entorno cambia.

El objetivo es otro: darte una entrada operativa mejor. Una forma de empezar bien, diagnosticar más rápido y decidir con menos fricción.

Lo que aprendí construyéndolo

Construir herramientas para uno mismo tiene un riesgo, que es sobreadaptarlas al flujo personal. Pero también tiene una ventaja enorme: si el dolor es real, la utilidad se prueba rápido. O te ahorra tiempo o no.

En este caso, lo valioso fue traducir molestias difusas en preguntas concretas: cómo ver rápido si el problema es general o local, cómo reducir el tiempo entre síntoma y causa probable, cómo rehacer una parte del sistema y cómo hacer que esa misma superficie sirva para agentes.

Cuando esas preguntas guían el diseño, el producto deja de ser una colección de features y empieza a tener una tesis.

Cómo arrancar

Si te suena familiar este problema, yo empezaría por la guía de instalación, seguiría con el repo y probaría el flujo de status y doctor antes de esperar a que algo se rompa. Es mejor conocer la herramienta en calma que descubrirla en medio de una falla.

FAQ

¿admirarr reemplaza las UIs de Plex, Radarr o Sonarr?

No. Funciona como capa operativa y de diagnóstico. Las interfaces siguen siendo útiles para tareas específicas y para inspección profunda.

¿sirve solo para Plex?

No. El enfoque es más amplio: stacks Plex o Jellyfin combinados con herramientas Arr y servicios asociados.

¿por qué un CLI y no otra dashboard web?

Porque una CLI bien diseñada reduce fricción, se integra mejor con automatizaciones y sirve tanto para humanos como para agentes.

¿puedo usarlo aunque mi stack ya esté funcionando?

Sí. admirarr no es solo para instalar. También sirve para observar, diagnosticar y recuperar control cuando el sistema ya está en producción personal.
